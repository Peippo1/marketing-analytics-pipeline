# Asset Prep

Use this folder guidance when preparing screenshots, a short GIF, or marketplace images for the repository.

## Recommended Asset Set

- `docs/assets/dashboard-home.png`
- `docs/assets/demo-outputs.png`
- `docs/assets/api-response.png`
- `docs/assets/quickstart.gif`
- `docs/assets/what-you-get.png`

## Capture Guidance

- Use the `make demo` flow first so outputs are predictable
- Capture the dashboard with realistic but non-sensitive sample data
- Keep terminal screenshots focused on the command and the successful result
- Avoid local usernames, machine names, and secret values in visible areas
- Use the same product wording that appears in the README and `PRODUCT.md`

## Suggested Visual Story

1. Dashboard hero and KPI area
2. Demo outputs folder with model and metrics files
3. FastAPI response example
4. One-command demo run in the terminal
5. A simple "what you get" summary image for listings
